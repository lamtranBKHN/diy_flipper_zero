## TgInitAsTarget ISO-DEP UID Emulation Feasibility

**PN532 supports ISO14443-4A target mode** via `TgInitAsTarget` with `SEL_RES=0x20`. The existing Type 4 NDEF emulation (`furi_hal_nfc_pn532.c:1476`) demonstrates this works.

**MfPlus/Ntag4xx UID-only emulation is feasible**: TgInitAsTarget with proper SENS_RES/SEL_RES/UID would make a reader detect the emulated UID. No protocol-level interaction (no AES/3DES crypto), but UID is visible. This is a future enhancement — requires new listener stubs that use TgInitAsTarget without full protocol emulation.

**DESFire UID-only emulation is also feasible** via same mechanism, but less useful since DESFire readers expect application-level interaction.

**PN532 hardware limitations for listener mode**:
- Only 3-byte NFCID1 (4th byte auto-generated by PN532)
- No Crypto1 engine (MF Classic emulation impossible)
- No AES/3DES engine (MfPlus/DESFire/NTAG4xx full emulation impossible)
- ISO-DEP framing handled in hardware (RATS/ATS/WTX automatic)

## SRIX Write Status

`srix_poller_write_block()` exists at `srix_poller.h:36` and is implemented at `srix_poller_i.c:76`. However, the protocol support file (`srix.c:121-125`) has `scene_write.on_enter = nfc_protocol_support_common_on_enter_empty` — no UI write handler. Adding `NfcProtocolFeatureWrite` without a handler would show a ghost "Write" button.

**Status**: Write API exists at poller level. UI handler needs implementation before enabling the flag.
