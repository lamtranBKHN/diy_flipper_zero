# NFC Tool Crash Analysis and Fix Plan

## Problem Summary

Device crashes during NFC Tool when reading MIFARE Classic card. Logs show:
- SAK=0x20, ATQA=0x4400 detected (ISO-DEP-only bank card, NOT MIFARE Classic)
- Scanner reports "MIFARE Classic" detected as child protocol
- Multiple `InCommunicateThru: status=0x01 (Timeout)` errors
- `Plugin timeout: cumulative 5000ms exceeded`
- `SPI held for 236ms (>100ms limit)` - ViewPort lockup
- Device crashes after transitioning to MF Classic system directory screen

## Root Cause Analysis

### The Issue: Missing Early Rejection in mf_classic_poller_detect()

The `mf_classic_poller_handler_detect_type()` (line 133-159 in mf_classic_poller.c) has an early rejection for ISO-DEP-only cards:

```c
// Early rejection: ISO-DEP-only cards (SAK bit 6 set, no MF Classic bits).
if(furi_hal_nfc_pn532_is_active() && (sak & 0x20) && !(sak & 0x18)) {
    FURI_LOG_D(TAG, "SAK 0x%02X: ISO-DEP only (no MF Classic bits), rejecting immediately");
    instance->state = MfClassicPollerStateFail;
    return command;
}
```

However, this check exists in the **handler** which runs AFTER detection. The `mf_classic_poller_detect()` function (line 2533) does NOT have this check.

### Crash Flow

1. Card placed (SAK=0x20, ATQA=0x4400 - ISO-DEP bank card)
2. Scanner detects ISO14443-3A and probes children protocols
3. `mf_classic_poller_detect()` called for `NfcProtocolMfClassic`
4. ATQA/SAK check fails (SAK=0x20 not in accepted list)
5. AUTH fallback attempted via `furi_hal_nfc_pn532_mf_auth()` (line 2562)
6. AUTH fails with InDataExchange status=0x13 (DEP bad data) - card is ISO-DEP, not MF Classic
7. Since `auth_fallback_attempted` is true, PN532 reset is triggered (line 2590-2597)
8. `furi_hal_nfc_pn532_reset()` stops IRQ thread and invalidates mode/tech
9. Subsequent child probes and supported card plugins hit this corrupted state
10. SPI I/O contention on shared I2C1 bus causes ViewPort lockups
11. Cumulative timeout cascade triggers device crash

## Solution

Add the same ISO-DEP-only early rejection check to `mf_classic_poller_detect()` function, BEFORE attempting the AUTH fallback.

### Change Required

**File:** `lib/nfc/protocols/mf_classic/mf_classic_poller.c`

**Location:** In `mf_classic_poller_detect()` function, after line 2553 (after primary ATQA/SAK check, before AUTH fallback at line 2559), add:

```c
// Early rejection: ISO-DEP-only cards (SAK bit 6 set, no MF Classic bits).
// This MUST be checked before AUTH fallback to avoid corrupting PN532 state.
if(furi_hal_nfc_pn532_is_active() && (sak & 0x20) && !(sak & 0x18)) {
    FURI_LOG_D(TAG, "SAK 0x%02X: ISO-DEP only (no MF Classic), rejecting in detect", sak);
    return false;
}
```

This prevents the AUTH fallback and PN532 reset on ISO-DEP bank cards, avoiding the crash cascade.

**Why check in both detect() and handler?** 
- The handler check (line 152) is defense-in-depth for ISO-DEP-only cards
- The detect() check prevents the AUTH fallback entirely - stopping the cascade before it starts
- SAK=0x20 (ISO-DEP) + ATQA=0x4400 is a bank card that will never respond to MF Classic auth

## Implementation Steps

1. **Add early rejection check** in `mf_classic_poller_detect()` (mf_classic_poller.c:2555)
2. **Verify build** with `./fbt` 
3. **Test with MF Classic card** - should work normally
4. **Test with bank card (SAK=0x20)** - should NOT crash, should skip MF Classic detection gracefully
