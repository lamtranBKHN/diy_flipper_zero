## New AGENTS.md content
```
## Build System
- Run `./fbt` from the repo root; it forwards to SCons.
- Load the toolchain: `scripts\toolchain\fbtenv.cmd` (Windows) or `source scripts/toolchain/fbtenv.sh` (Unix). Shortcut: `./fbt -s env` dumps env variables.

## Target
- DIY board uses STM32WB55CGU6, default target `f7` (`TARGET_HW=7`). CI builds only the `f7` matrix.

## Core Commands
- `./fbt` – debug firmware (no external apps).
- `./fbt fap_{APPID}` – build external app (e.g. `./fbt fap_qrcode`).
- `./fbt launch APPSRC=applications_user/<path>` – build and run an app on the device.
- `./fbt flash` – flash via ST‑Link / Black Magic probe.
- `./fbt flash_usb_full` – build + flash over USB (device must be in DFU mode).
- `./fbt copro_dist` – package Core2 BLE radio stack.
- `./fbt updater_package` – create OTA update bundle.

## Lint / Format
- `./fbt lint_all` – Python (`black --check`), C/C++ lint, image lint.
- `./fbt format_all` – `black`, `clang-format`, image formatting.
- Individual targets: `lint_py`, `format_py`, `lint_img`, `format_img`.

## Debugging
- `./fbt debug` – GDB via OpenOCD (default probe).
- `./fbt blackmagic` – GDB via Black Magic probe.
- `./fbt debug_other` – attach GDB to an arbitrary ELF.
- OpenOCD config is defined in `fbt_options.py:74-83`.

## Key Options
- `FIRMWARE_APP_SET=unit_tests` – include unit‑test suite.
- `COMPACT=1` – size optimisation.
- `DEBUG=0` – strip debug symbols.
- `FBT_NO_SYNC=1` – skip automatic submodule updates.
- `VERBOSE=1` – enable verbose SCons output.

## Hardware Quirks
- **PN532 NFC**: I2C1 (SCL PA9, SDA PB9), 7‑bit address `0x24` (`PN532_I2C_ADDR_7BIT`), enabled via `PN532_ENABLED` in `targets/f7/boards/custom_pn532_board.mk`.
- **PCF8574 I/O expander**: I2C1, address `0x20`; buttons on P0‑P5, vibro on P6, buzzer on P7, INT pin on PB0.
- **CC1101**: SPI1, CS = PA15, G0 = PA1.
- **SD card**: SPI1, CS = PA10.
- **OLED**: SPI1 (SH1106/SSD1306).
- **I2C3** (PA7/PB4) disabled – pins used for other functions.
- Pin macros are defined in `targets/f7/furi_hal/furi_hal_resources.h`.

## Gotchas
- `./fbt` auto‑updates git submodules on first run; set `FBT_NO_SYNC=1` to prevent.
- `flash_usb_*` requires DFU mode and a correct USB driver (Zadig on Windows).
- All `fbt` aliases are defined in `SConstruct`, not as separate scripts.
- CI enforces API‑symbol consistency: `targets/f7/api_symbols.csv` must match the upstream OFW channel.
```

## New Plan: Verify NFC Tool Stability

### Goal
Ensure the NFC HAL and PN532 driver operate without crashes or errors during all supported operations (Read, Write, Emulate) on the DIY board.

### Steps
1. **Code Survey**
   - Use `cavecrew-investigator` to list all files containing `ST25R3916` references and any unconditional includes.
   - Use `cavecrew-investigator` to list all functions in `furi_hal_nfc_*` that call `furi_check` or `furi_assert`.
2. **Conditional Compilation**
   - Guard all ST25R3916‑specific code with `#if defined(FURI_HAL_NFC_ST25R3916)` or similar.
   - Verify `FURI_HAL_NFC_PN532_ONLY` is defined appropriately in `targets/f7/furi_hal/furi_hal_nfc.c`.
3. **Error‑Handling Review**
   - Replace any `furi_check` that aborts on PN532‑only hardware with proper error‑return checks.
   - Ensure all mutex acquire/release paths are balanced; no early returns without release.
4. **PN532 Driver Validation**
   - Confirm I2C address (`0x24` 7‑bit) and pins (SCL PA9, SDA PB9) in `furi_hal_i2c_config.c`.
   - Add retry logic for I2C NACKs if not present.
   - Verify that all I2C reads request only required length (avoid 270‑byte reads).
5. **Unit / Integration Tests**
   - Add/extend unit tests under `applications_user/nfc_test/` exercising:
     * `furi_hal_nfc_poller_read`
     * `furi_hal_nfc_poller_write`
     * `furi_hal_nfc_emulate`
   - Build with `FIRMWARE_APP_SET=unit_tests` and run `./fbt unit_tests`.
6. **Runtime Verification**
   - Build debug firmware (`./fbt DEBUG=1`) and flash to board.
   - Run NFC demo apps (reader, writer, emulator) while attached to GDB (`./fbt debug`).
   - Observe logs for any aborts, asserts, or unhandled errors.
7. **Static Analysis**
   - Run `./fbt lint_all` to catch missing error checks.
   - Use `grep` to find any remaining `#error` or `#warning` that could stop compilation on PN532‑only builds.
8. **Documentation Update**
   - Update `README.md` and hardware notes with PN532‑only limitations and required I2C configuration.
   - Mark unsupported features (listener mode, ISO15693, etc.) as “Not supported on DIY board”.
9. **CI Integration**
   - Add a CI job that builds the firmware with `PN532_ENABLED` and runs the new NFC unit tests.
   - Ensure the job fails on any crash or unexpected return code.

### Acceptance Criteria
- All NFC demo applications run to completion without aborts.
- Unit tests covering read/write/emulate pass on the DIY board.
- No `furi_check` aborts triggered during NFC operations.
- CI passes with the new NFC verification job.
