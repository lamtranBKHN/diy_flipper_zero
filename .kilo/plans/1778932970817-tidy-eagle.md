# Plan: Fix MIFARE Classic Dictionary Attack Slowness

**Date:** 2026-05-16
**Goal:** Fix CRC stripping bug in `furi_hal_nfc_pn532_exchange_internal` that causes every MIFARE Classic auth command to fail with CRC error (status `0x02`), resulting in 2+ hour dictionary attacks.

---

## Root Cause

In `furi_hal_nfc_pn532_exchange_internal()` (`targets/f7/furi_hal/furi_hal_nfc_pn532.c:492`):

1. `furi_hal_nfc_pn532_tx()` (line 830-832) calls `exchange_internal` with `strip_crc_from_tx=true`
2. `exchange_internal` receives `tx_bytes = [0x60, block, crc_l, crc_h]` (4 bytes with CRC from iso14443_3a poller)
3. Lines 504-507: CRC is stripped from `effective_tx_len` (4 → 2) and `send_len` (2)
4. Lines 676-694: Auth command detected via `send_len >= 2 && tx_bytes[0] == 0x60`
5. Line 691: `raw_auth[2] = {tx_bytes[0], tx_bytes[1]}` — only 2 bytes, CRC is LOST
6. Line 692-693: `InCommunicateThru` sends `[0x60, block]` (no CRC) to card
7. Card rejects frame → PN532 returns status `0x02` (CRC error)
8. → auth fails → `target_valid = false` → 300ms re-poll → next key... × 3000+ keys

### Key Insight

- `InDataExchange` (normal path, line 786-794): uses `ide_len = tx_len` (FULL length with CRC) — PN532 strips CRC internally. **Correct.**
- `InCommunicateThru` (auth path, line 690-693): sends raw bytes — PN532 does NOT add CRC. Must include CRC in payload. **Bug: CRC was stripped.**

---

## Fix

**File:** `targets/f7/furi_hal/furi_hal_nfc_pn532.c`, lines 689-694

**Change:** Send `tx_bytes` with `tx_len` (includes CRC) instead of `raw_auth[2]` (stripped CRC):

```c
// Before (broken):
uint8_t raw_auth[2] = {tx_bytes[0], tx_bytes[1]};
err = furi_hal_pn532_in_communicate_thru(
    raw_auth, sizeof(raw_auth), rx_payload, sizeof(rx_payload), &rx_len);

// After (fixed):
err = furi_hal_pn532_in_communicate_thru(
    tx_bytes, tx_len, rx_payload, sizeof(rx_payload), &rx_len);
```

### Rationale

- `tx_bytes` is the original buffer passed to `exchange_internal`
- `tx_len` is the original length including CRC (4 bytes for `[0x60, block, crc_l, crc_h]`)
- `InCommunicateThru` sends raw bytes without modification — CRC must be present
- Backdoor auth path (0x64/0x65, line 678-688) is **unaffected** — it builds its own 9-byte command buffer with key data, doesn't use `tx_bytes` for the payload
