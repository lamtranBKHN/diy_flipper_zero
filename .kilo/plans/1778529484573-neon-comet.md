# Plan: PN532-Only NFC HAL Cleanup

## Goal
Strip all ST25R3916 code from the NFC HAL layer — make the firmware build and run with PN532 as the sole NFC chip. PN532 is on I2C1 (PA9 SCL, PB9 SDA) with **no IRQ/INT pin** — all communication is polled I2C.

## Current State
- `custom_pn532_board.mk` already defines `FURI_HAL_NFC_ST25R3916=0` and `PN532_ENABLED` globally
- BUT: `#ifdef FURI_HAL_NFC_ST25R3916` in headers (`furi_hal_nfc_i.h`, `furi_hal_nfc_irq.c`) evaluates to TRUE (macro is defined, even as 0), so ST25 driver headers get included
- Each tech `.c` file has redundant `#ifndef`/`#define FURI_HAL_NFC_ST25R3916 0` boilerplate
- ST25 code is guarded by `#if FURI_HAL_NFC_ST25R3916` → dead at value 0, but still compiled (headers, syntax)
- PN532 paths are guarded by `if(furi_hal_nfc_pn532_is_active())` — these are live
- `furi_hal_spi_bus_handle_nfc` is **not initialized** in `furi_hal_spi_config.c` (commented-out declaration + commented-out init) — already dead
- The NFC IRQ GPIO (`gpio_nfc_irq_rfid_pull`) is aliased to PCF8574_INT pin — only relevant for ST25
- Timers TIM1/TIM17 are still used by PN532 paths for FWT timeout events

## Step 1: `custom_pn532_board.mk` — Remove ST25R3916 define
Remove `FURI_HAL_NFC_ST25R3916=0` from the CPPDEFINES. Since we're stripping all ST25 code, this macro should not exist anywhere.
- Keep `PN532_ENABLED` only

Decision: keep declarations in `furi_hal_nfc_i.h` since they're still real functions (just PN532-stubbed).
