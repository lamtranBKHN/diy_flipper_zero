### NOT IMPLEMENTED / DEFERRED

| Item | Reason |
|------|--------|
| ISO15693 re-enable in scanner | PN532 does NOT support ISO15693 (NXP PN532 datasheet — no Vicinity card support) |
| FeliCa read/write poller changes | Already exists in `felica_poller.c`; works through auto-poll + InDataExchange. No changes needed. |
| NTAG FAST_READ auto-integration | Standard READ (0x30) already returns 4 pages/call. FAST_READ benefit for polling is marginal. API function added for manual use. |
| PN532-specific listener protocol config | TGINITASTARGET uses hardcoded UID (all zeros), SENS_RES (0x04 0x00), SEL_RES (0x20). Should be configurable from protocol listener data. |
| InAutoPoll for multi-protocol scan | Could reduce I2C exchanges but requires scanner restructuring. Low priority. |

### ESP32 Port Reference Analysis

Two reference projects were analyzed for robustness improvements:

| Project | Key Findings | Adopted |
|---------|-------------|---------|
| **ESP32 port** (`Flipper-Zero-ESP32-Port`) | Listener abort slices (200ms polling), PN532-native MIFARE Classic auth | Listener abort slices implemented |
| **Full firmware** (`firmware/`) | Same architecture as our codebase; identical NFC protocol layer | No changes needed (architecture matches) |

**Improvements from ESP32 port NOT adopted:**
- PN532-native MIFARE Classic auth with Crypto1 fallback: Our code uses 2-byte pass-through InDataExchange which also works (PN532 forwards auth to card; host handles Crypto1 in software). Functionally equivalent.
- User-configurable MIFARE keys: Nice-to-have, not a functional gap
- NDEF URL abbreviation table: Simple but low priority
- EMV TLV fallback parsing: Only relevant for banking card reading, not core functionality

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| Auto-poll fires unexpectedly after comm failure | Guard: `effective_tx_len < 3` returns error |
| Listener mode conflicts with poller mode | `reset()` clears `listener_active` + sends InRelease before switching modes |
| I-block wrapping incorrect for 4-layer decoder | Uses PCB=0x02 (I-block, seq=0, no CID/NAD) — matches existing 4-layer decoder expectations |
| 4A listener I-block activation breaks ST25 path | Only triggers on I-block PCB (bits 7:6=00) — RATS (0xE0) never matches this. ST25 path unchanged. |
| WAIT_FOREVER timeout passes through unclamped | PN532 SAM timeout (~1s) limits actual wait regardless of host timeout value |
