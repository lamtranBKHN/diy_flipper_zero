# NFC Subsystem Debugging Plan

**Goal**: Fully audit, debug, and improve the NFC stack, driver, and UI for the DIY Flipper Zero (STM32WB55, PN532 I2C). The plan covers static analysis, concurrency checks, protocol support verification, memory management, error handling, instrumentation, testing, and documentation.

---

## 1. Component Inventory
- **HAL driver**: `targets/f7/furi_hal/furi_hal_nfc*.c` and corresponding headers.
- **Protocol implementations**: `lib/nfc/protocols/` (`nfc_iso14443a.c`, `nfc_iso14443b.c`, `nfc_felica.c`, `nfc_iso15693.c`).
- **Generic NFC stack**: `lib/nfc/` (`nfc.c`, `nfc_scanner.c`, `nfc_poller.c`, `nfc_listener.c`, `nfc_device*.c`).
- **Test harness**: `test_nfc/` (`nfc_test_app.c`, `nfc_test.c`).
- **UI scenes**: `applications/main/nfc/scenes/` (read/write, emulation, tag‑specific scenes).
- **Resources**: sample `.nfc` files under `resources/nfc/`.
- **Build system**: `./fbt` targets for `fbt nfc_test` and apps.

---

## 2. Static Code Analysis
1. **Search for markers**:
   - Run `grep` for `TODO`, `FIXME`, `BUG`, `NOTE` across all NFC‑related files.
2. **Check mutex usage**:
   - Find all calls to `furi_hal_nfc_acquire()` / `release()`.
   - Verify each wrapper (`poller_tx`, `listener_tx`, etc.) does **not** reacquire a lock already held (the recent race bug).
3. **Validate error propagation**:
   - Grep for patterns where a function call result is ignored (`;` without `if`).
   - Ensure every API that returns `FuriHalNfcError` is checked.
4. **Memory allocation audit**:
   - Search for `furi_alloc`, `malloc`, `calloc` in NFC files.
   - Verify a matching `furi_free`/`free` on every path, especially in error branches.
5. **Protocol guard checks**:
   - Ensure all PN532‑only code sections are wrapped with `#ifndef FURI_HAL_NFC_PN532_ONLY` where appropriate (e.g., listener mode functions).
6. **Logging completeness**:
   - Look for missing `FURI_LOG_*` statements in critical paths (initialisation, field on/off, protocol switches).

---

## 3. Concurrency / Race Condition Review
- **Lock boundaries**: Map call graph from public API (`furi_hal_nfc_*`) down to low‑level PN532 driver. Confirm lock is held exactly once per transaction.
- **Interrupt safety**: Verify any ISR (`furi_hal_nfc_irq.c`) does not touch shared state without disabling interrupts or acquiring the mutex.
- **Thread safety of UI callbacks**: Confirm UI scenes do not call NFC functions from background threads without proper synchronization.

---

## 4. Protocol Support Verification
| Protocol | Expected Support | Current Implementation | Gaps |
|----------|------------------|-----------------------|------|
| ISO14443‑A | Full (including anti‑collision, UID, SAK, ATQA) | `furi_hal_nfc_iso14443a.c` | Verify listener guard & field control.
| ISO14443‑B | Full | `furi_hal_nfc_iso14443b.c` | Confirm PN532 fallback handling.
| FeliCa   | Full | `furi_hal_nfc_felica.c` | Same as above.
| ISO15693 | Full (optional) | `furi_hal_nfc_iso15693.c` | PN532 does **not** support – should return `NOT_SUPPORTED` cleanly.
| MF Classic / Ultralight / Desfire | Implemented in lib/nfc protocols & UI scenes | Various `nfc_scene_*` files | Ensure edge‑case handling (e.g., wrong UID length).

**Actions**:
- Write a small script (or manual review) to invoke each `nfc_poller_` entry point with a mock device (`nfc_mock.c`) to confirm return codes for unsupported protocols.
- Verify `nfc_scanner.c` enumerates all protocols when PN532 is present.

---

## 5. Listener / Emulation Path
- Confirm that **any** call to `furi_hal_nfc_set_mode(FuriHalNfcModeListener)` is guarded by `#ifndef FURI_HAL_NFC_PN532_ONLY`.
- Review UI scenes that offer “Emulate Card” – they must disable the option when PN532 is the only chip.
- Add unit‑test to `nfc_test` for listener mode on PN532 (should return `FURI_HAL_NFC_ERROR_NOT_SUPPORTED`).

---

## 6. Memory Management & Buffer Handling
- Identify all static buffers (e.g., the 270‑byte I2C read buffer). Evaluate if they can be reduced safely.
- Check for stack overflows: large local arrays in protocol parsers.
- Add `FURI_LOG_E` on allocation failures.
- Run `valgrind`‑style checks on host‑simulated code (if possible) or enable `__ARMCC_VERSION` memory guards.

---

## 7. Error Handling & Return‑Code Consistency
- Define a matrix of functions → possible error enum values.
- Ensure every public API translates lower‑level errors to a consistent `FuriHalNfcError`.
- Add missing `furi_check` asserts (e.g., the recent `low_power_mode_stop` bug).

---

## 8. Instrumentation & Logging Enhancements
- Insert `FURI_LOG_D` at entry/exit of each major NFC function (poller, listener, timer, irq).
- Include timestamps for field on/off events.
- Optionally enable a compile‑time flag `NFC_DEBUG_TRACE` to dump raw I2C frames.

---

## 9. Unit‑Test Coverage Expansion
1. **Existing tests**: Review `test_nfc/nfc_test.c` – note which protocols are exercised.
2. **Add tests** for:
   - Unsupported protocol handling (ISO15693 on PN532).
   - Listener mode rejection on PN532.
   - Field control no‑ops on PN532.
   - Buffer overflow protection.
3. **Mock driver**: Extend `nfc_mock.c` to simulate error conditions (e.g., I2C NACK, timeout).
4. **CI integration**: Ensure `./fbt unit_test` runs these new tests.

---

## 10. Functional / Hardware Validation
1. **Build test app**: `./fbt fap_nfc_test` (or `./fbt test_nfc`).
2. **Flash**: `./fbt flash` to device.
3. **Run scenarios**:
   - Read a MIFARE Classic tag, verify UID, SAK, block reads.
   - Write to an Ultralight tag, check data persistence.
   - Attempt to emulate a tag, ensure device reports `NOT_SUPPORTED` on PN532.
   - Scan for all supported protocols, confirm UI lists them correctly.
4. **Capture logs**: Use `log level debug` to obtain detailed trace, compare against expected sequence.
5. **Performance check**: Measure time for a full ISO14443‑A read; confirm the 270‑byte read inefficiency is acceptable or reduce it.

---

## 11. UI Review
- Walk through each `nfc_scene_*.c` file:
  - Verify error dialogs map to correct NFC error codes.
  - Ensure UI disables unsupported options (e.g., Listener, Emulate) on PN532 builds.
  - Confirm that “Save” actions correctly free allocated buffers.
- Add unit tests for UI callbacks where feasible (using the Flipper UI test harness).

---

## 12. Regression & CI Verification
- After each fix, run the full CI pipeline:
  - `./fbt lint_all`
  - `./fbt format_all`
  - `./fbt unit_test`
  - `./fbt fap_dist` (build all NFC‑related apps).
- Ensure no new warnings appear in `furi_hal_nfc*` or `lib/nfc/*`.

---

## 13. Documentation Update
- Update `README`/`docs` describing NFC capabilities on DIY board.
- Clearly list PN532 limitations (no listener mode, field control no‑ops, supported protocols).
- Add a troubleshooting section for common failures (e.g., I2C bus desync, 270‑byte read performance).
