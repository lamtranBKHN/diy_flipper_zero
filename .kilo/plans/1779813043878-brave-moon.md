# Plan: NFC Feature Consolidation and Verification

## Crash Analysis: MIFARE Classic "System Directory" Read Crash

### Root Cause Investigation
The "system directory" read crash occurs in `mf_classic_poller.c` during nested dictionary attack path:

**Primary Crash Path (Memory):**
- Line 1260-1266: `add_nested_nonce()` allocates/reallocates nonce array
- Line 1886: "Failed to add nested nonce to array. OOM?" logged on failure
- Line 2298-2322: On OOM, nonces freed and retry counter incremented
- If repeated OOM from fragmented heap + I2C contention → heap exhaustion → crash

**Secondary Crash Path (I2C Contention):**
- PN532 uses I2C1 shared with OLED display (PB6) on DIY board
- `NFC_SCANNER_INTER_PROBE_YIELD_MS=20` prevents GUI starvation (line 19)
- Without yield, 270-byte I2C reads starve display thread → ViewPort lockup warnings (AGENTS.md)

**System Directory Specific:**
- Lines 2154-2168: `mf_classic_system_dict` loads from `assets/mf_classic_dict_nested.nfc`
- If file missing or corrupted, `keys_dict_alloc()` returns NULL
- Handler continues but log shows: "System dict failed, using fallback path"

### Fixes Already Implemented (AGENTS.md verified)

1. **Fail Retry Limit (mf_classic_poller.c:2384-2402)**
   - Added `fail_retry_count` field, max 5 iterations before stop
   - Prevents infinite loop → watchdog crash

2. **PN532 Native Auth Recovery (furi_hal_nfc_pn532.c:1742-1799)**
   - `furi_hal_nfc_pn532_mf_deauth()` sets `needs_relist=true`, `target_tick=0`
   - Forces re-poll on next auth attempt after 0x14 failure

3. **I2C Contention Mitigation (nfc_scanner.c:15-19)**
   - Increased idle delay from 50ms to 100ms
   - Added 20ms yield between probes for GUI thread

4. **Jewel/Topaz Protocol (jewel_poller.c + nfc_scanner.c:117-119)**
   - Added to PN532 probe order for NTAG-like detection
